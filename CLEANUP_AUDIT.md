# Ratatoskr Over-Engineering & Design Audit (ponytail)

> One-shot repo-wide audit. **Lists findings; applies nothing.** Generated by a
> 33-agent fan-out (17 subsystem finders → 17 verifiers that grep-confirmed every
> deletion/single-caller claim against the real tree before it landed here).
>
> Scope: complexity & design-correctness only. Correctness bugs, security, and
> performance are explicitly out of scope — run a normal review pass for those.

The codebase is a **single-owner Telegram bot carrying ~186k LOC of app code and
~134k LOC of tests** — and it shows the scars of a DDD re-layering + a
ChromaDB→Qdrant migration that were never cleaned up after. Roughly **a quarter
of the application is removable without changing a single user-visible behavior.**
Nothing here is a "nice idea for later" — every confirmed item is dead, a
single-caller wrapper, or a hand-rolled copy of something stdlib/Pydantic/SQLAlchemy
already ships.

## The 8 themes (ranked by impact)

### 1. DDD ports/adapters/DTO layering with one implementation each — **~3,500L**

The single largest cluster. Hexagonal scaffolding around a personal bot that has
exactly one concrete implementation of everything.

- `app/application/ports/` — **41–44 `Protocol` Port classes** wrapping repos that have one impl (`~2,000L`).
- DTO triple-mapping: `summary_dto.py` + `request_dto.py` with `to/from_domain_model` (`237L`), used only by tests.
- `app/di/repositories.py` — 19 `build_X_repository` factories, each `return XAdapter(...)` (`80L`).
- `app/api/dependencies/database.py` — 17 repo-factory functions duplicating the same wiring (`80L`).
- Passthrough services: `SummaryService` (`172L`), `SyncService`→`SyncFacade` double wrapper (`200L+`), `DigestFacade` (`130L`).
- `app/domain/models/request.py` — 16 instance methods (`mark_as_*`, `is_*`) all unused (`150L`).
- `GetUnreadSummariesUseCase` maps repo dicts → domain → back (`70L`).

### 2. Dead / superseded subsystems left by the migration — **~3,000L+**

- **Domain event-bus messaging** (`app/infrastructure/messaging/`): `event_bus.py` + `wiring.py` + 11 handlers (`1,757L`), driven by domain events that **are subscribed but never published** (`request_events/tag_events/rule_events` — `330L`). Only `SummaryMarkedAsRead` is ever published, and its handler just re-persists what mark-as-read already wrote.
- `migrate_vector_store.py` — finished ChromaDB→Qdrant one-shot CLI; **no `chromadb` dep or `chroma` reference exists anymore** (`458L`).
- `commands.py` — old Telegram command framework, superseded by `command_dispatch/` (`234L`).
- `db/batch_operations.py` + `db/query_profiler.py` — imported only by their own tests (`356L`).
- `cache/batch_progress_cache.py` + `cache/query_cache.py` — zero / test-only callers (`545L`).
- `firecrawl/exceptions.py` 8-class hierarchy + `openrouter/exceptions.py` subclasses — never raised (`330L`).
- `db/runtime/operation_executor.py` + `DatabaseExecutorPort` — callers already use `Database.session()` directly (`210L`).

### 3. Alternate LLM providers that are never used — **~1,700L**

`app/adapters/llm/openai/`, `anthropic/`, `ollama` branches, `message_sanitizer.py`,
their config classes, and the `LLMClientFactory` branches. In practice everything
goes through OpenRouter. (`yagni` — also a candidate to drop the `openai`/`anthropic`
SDKs from deps if they're only here for this.)

### 4. Test bloat — the largest line category — **~9,000L+**

`tests/` is bigger than `app/`. Verified cuttable:

- 9 "coverage-gaming" test files whose docstrings cite source line numbers / uncovered branches (`4,500L`).
- `tests/architecture/test_lint_rules.py` + closure/mutability tests that **shell out to `ruff` to re-test what ruff + CI already enforce** (`1,070L`).
- `test_ports_compliance.py`, `test_tier2_direct_module_coverage.py`, exhaustive `test_telegram_models.py`, dead `tests/characterization/`.
- *(Plus the git_backup test footprint — see Uncertain, `~3,000L`.)*

### 5. Multi-tenant / collaboration machinery in a single-owner bot — **~650L**

- Collections sharing/invite/ACL: router (`95L`) + repository (`250L`) + `CollectionService` RBAC (`130L`).
- `AccessController` failed-attempt/stranger-defense system (`150L`).
- Dead per-request user-scoping in `mcp/context.py` (`30L`).

### 6. `git_backup` over-tooling ported wholesale from "gitout" — **~900L**

For a default-**off**, single-user backup: Healthchecks.io pings, CSV/JSONL metrics
export, a configurable `RetryPolicy`/`BackoffStrategy` engine, a 9-category
`ErrorCategory` taxonomy, and a `_preflight_storage_check`. A flat
"list due → `git remote update --prune` under a semaphore" loop covers it.

### 7. Config declared ~4× + hot-reload nobody triggers — **~380L**

- `config_holder.py` `ConfigReloader` background poll loop for runtime model hot-reload (`190L`).
- `load_models_yaml()` duplicates `load_ratatoskr_yaml()` YAML parsing (`100L`).
- `AppConfig` schema restated across settings layers (`90L`).

### 8. Sync subsystem over-built — **~270L**

`SyncService`/`SyncFacade` double wrapper (theme 1), the pluggable
`SyncEntityAdapter` framework (`180L`), `_Null*Repository` objects, and datetime→ISO
coercion reimplemented 3× (`serializer.py`).

## Caveats — verify before deleting

- **H17 `WebwrightEnricher` ("Path C")** contradicts `CLAUDE.md`, which documents it
  as invoked by the summarize loop, and the `LLMCall.attempt_trigger` enum has a
  `webwright_tool` value for it. The verifier found no live caller — but confirm by
  hand, and if you delete it, also update `CLAUDE.md`, `docs/explanation/webwright.md`,
  and drop the dead enum value. (`/browse` Path B and the scraper-chain rung are
  separate and stay.)
- **H27 scraper `getattr` defaults** already diverge from the Pydantic `Field`
  defaults in 2 places (`defuddle_api_base_url`, `scrapegraph_enabled`) — switching
  to direct attribute access changes nothing at runtime *today* but removes a real
  latent misconfiguration trap. Double-check those two fields' intended defaults.
- Several deletions have companion test files and **doc entries** (e.g. H7
  `migrate_vector_store` is referenced in `docs/reference/cli-commands.md`). Remove
  docs in the same commit.
- Any model/enum removal needs an Alembic migration if it touches DB-backed enums.

## Suggested execution order

**Tranche A — pure dead code (safe, high-confidence, ~5–6k LOC).**
Re-export shims, zero-caller modules, never-published events + the messaging bus,
finished migration CLI, test-only modules. Each is a delete + its test + any doc
line. Low risk, mechanical, big win. (H2, H6, H7, H9, H10, H12, H13, H14, H16, low-tier shims.)

**Tranche B — collapse single-impl layers (behavior-preserving refactor, ~6–8k LOC).**
DTO mappers, passthrough services, port protocols, repo factories, domain-model
methods. Needs care + the test suite green after each step. (Themes 1, 8 + H19–H23, H28–H31.)

**Tranche C — judgment calls (the Uncertain table).**
Decide per-item whether the seam earns its keep (ports abstraction, auth file split,
chat-path split, git_backup test depth).

**Test cleanup (Tranche A-tests, ~9k LOC)** can run in parallel with A — the
coverage-gaming and ruff-reimplementing tests (H1, H4) are independent of app code.

## Execution status (updated after the cleanup run)

Executed on branch `cleanup/ponytail-audit` in 7 verified commits — **~16k lines
removed**, ruff fully clean, ~5.3k tests still collect, mypy clean on every
touched module, `app.api.main` + `app` import cleanly. (No local Postgres, so
DB/integration tests were not run — CI is the final gate for those.)

**Done:** dead agents, the entire event-bus messaging subsystem + unpublished
events, finished migration CLI, dead caches/profilers, unused openai/anthropic/
ollama LLM providers, dead exception hierarchies, the old telegram command
framework, ~2k lines of coverage-gaming/meta tests, re-export shims, stdlib
reinventions (numpy cosine, os.path.commonprefix), DTO mappers, passthrough
services (SummaryService, SyncService), duplicate repo-factory + task wiring,
config hot-reload loop, the 89 scraper `getattr` defaults (incl. 2 that silently
diverged), provider-tiering de-drift, dead rate-limiter/MCP-scope/bootstrap code,
and the dead `WebwrightEnricher` (Path C). Docs (`CLAUDE.md`, `AGENTS.md`,
`webwright.md`) synced; two obsolete event-bus task notes removed.

**Verify caught the audit being wrong (kept):** `summary_validator`,
`DatabaseExecutorPort` (wired live), `RerankingService` (wired), `VectorStore`
protocol (shared exceptions), the 8 formatting protocols (real polymorphism),
two-layer `SearchFilters`, `token_crypto`, `circuit_breaker`, twitter
`url_patterns`, all git_backup ops tooling (config-gated but live), and
`load_models_yaml` (not a true duplicate).

**Deliberately DEFERRED (needs CI-green + review, not done):**

- **`application/ports` collapse (~2000L)** — a second audit agent flagged this
  same target as "keep the seam"; collapsing 41 Port protocols touches the whole
  app and can't be verified without the integration suite. Decide the seam first.
- **Collections collaboration/ACL removal (~475L)** — changes the OpenAPI
  contract + may affect the web frontend/sync; needs `make generate-openapi` +
  frontend check.
- **`summary_contract_impl` normalization redesign (H11) + 8-module shrink (~400L)**
  — behavior-sensitive validation pipeline; needs the full (partly DB-backed)
  summary suite.
- **`DigestStore` async/sync dual-API (H21, ~130L)** — the sync wrappers have
  real sync callers; collapsing needs DB tracing.
- **Reshape-not-delete design items** (AttachmentProcessor 4-file split, LLM
  clients copy-vs-inherit, YouTube media flow, background executor collaborators,
  ResponseFormatter facade, ~430L) — lower value, behavior-sensitive.
- **The 17 "uncertain" findings** — out of scope for this run (judgment calls).
- Left the Postgres `attempt_trigger.webwright_tool` enum value in place
  (removing an enum value needs a migration for zero benefit); marked reserved.
